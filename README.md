# StockIntegration-ISI-TP2

Projeto de Integração de Sistemas de Informação que implementa um sistema completo de gestão de stock através de vários serviços e tecnologias de integração. O projeto inclui uma API REST, uma API SOAP, uma aplicação cliente desenvolvida em **Windows Forms** e uma base de dados SQL que suporta todas as operações de consulta e atualização de stock. O objetivo central passa por demonstrar e comparar diferentes abordagens de integração (REST vs SOAP), garantindo interoperabilidade e comunicação entre sistemas distintos.

## Funcionalidades principais

* **API REST (RESTfullStock)**
  * Endpoints RESTful para gestão de produtos e stock.
  * Operações CRUD completas.
  * Respostas em formato JSON.
  * Arquitetura simples e moderna para serviços web.

* **API SOAP (StockSOAP)**
  * Webservice SOAP para gestão de stock.
  * Comunicação baseada em XML.
  * Métodos equivalentes à API REST para efeitos comparativos.
  * Demonstração de integração tradicional orientada a contratos.

* **Aplicação Cliente (StockClient)**
  * Desenvolvida em **Windows Forms** (primeira experiência com esta tecnologia).
  * Capacidade de consumir **serviços REST e SOAP**.
  * Interface gráfica para consulta e gestão de stock.
  * Comparação prática de tempos de resposta e formatos de comunicação.

# StockIntegration-ISI-TP2

Information Systems Integration project implementing a complete stock management system through multiple integration technologies. The solution includes a REST API, a SOAP API, a client application built using **Windows Forms**, and a SQL database supporting all stock operations. The main goal is to demonstrate and compare different integration approaches (REST vs SOAP), ensuring interoperability between heterogeneous systems.

## Main Features

* **REST API (RESTfullStock)**
  * RESTful endpoints for product and stock management.
  * Full CRUD operations.
  * JSON-based responses.
  * Modern and lightweight service architecture.

* **SOAP API (StockSOAP)**
  * Traditional SOAP webservice for stock management.
  * XML-based communication.
  * Methods equivalent to the REST API for comparative analysis.
  * Demonstrates contract-based service integration.

* **Client Application (StockClient)**
  * Developed using **Windows Forms** (first experience with this technology).
  * Consumes both **REST and SOAP services**.
  * Graphical interface for stock visualization and management.
  * Practical comparison of response formats and performance.
